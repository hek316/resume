# Plan: 개발자 이력서 (Developer Resume)

## 1. 개요

| 항목 | 내용 |
|------|------|
| Feature | 4년차 백엔드 개발자 이력서 웹사이트 |
| 목표 | 심플하고 모던한 디자인의 웹 이력서. 면접관이 빠르게 핵심 역량을 파악할 수 있도록 가독성 극대화 |
| 대상 사용자 | 채용 담당자, 면접관, 헤드헌터 |
| 프로젝트 레벨 | Starter (정적 웹사이트) |

## 2. 핵심 요구사항

### 2.1 디자인 방향
- **심플 & 모던**: 불필요한 장식 없이 깔끔한 레이아웃
- **가독성 우선**: 면접관이 30초 안에 핵심 정보를 파악할 수 있는 구조
- **프로페셔널**: "일 잘하는 사람"이라는 인상을 주는 정돈된 구성
- **반응형**: 모바일/태블릿/데스크톱 대응

### 2.2 이력서 섹션 구성

| 순서 | 섹션 | 설명 | 우선순위 |
|------|------|------|----------|
| 1 | **헤더/프로필** | 이름, 직함(Backend Developer), 핵심 연락처 | P0 |
| 2 | **경력 요약 (Career Summary)** | 3~4줄 핵심 요약. 경력 연차, 전문 분야(Backend, Migration, Performance), 핵심 성과 1줄. 면접관이 가장 먼저 읽는 영역 | P0 |
| 3 | **핵심 역량 요약** | 기술 스택 뱃지 + 한줄 요약 | P0 |
| 4 | **경력 사항** | 회사명, 기간, 역할, 주요 성과 (성과 중심으로 기술) | P0 |
| 5 | **프로젝트** | 주요 프로젝트 카드형 레이아웃 (기술스택 태그 포함) | P0 |
| 6 | **기술 스택** | 카테고리별 스킬 레벨 시각화 (Language, Framework, DB, Infra 등) | P1 |
| 7 | **학력/자격증** | 간결하게 나열 | P2 |
| 8 | **링크** | GitHub, Blog, LinkedIn 등 | P1 |
| - | **PDF 다운로드 버튼** | 헤더 영역에 PDF 다운로드 버튼 배치. 브라우저 print 기반 PDF 생성 | P0 |

### 2.3 디자인 원칙
- **컬러**: 모노톤 베이스 + 포인트 컬러 1개 (블루 계열 추천)
- **타이포그래피**: 한글 - Pretendard / 영문 - Inter (모던하고 가독성 좋은 폰트)
- **레이아웃**: 1컬럼 중심, 넓은 여백, 섹션 간 명확한 구분
- **인터랙션**: 애니메이션 없음. 정적인 문서 느낌 유지
- **AI 티 제거**: 이모지/아이콘 사용 금지, 뱃지/태그 스타일 금지, 과도한 장식 요소 금지. 플레인 텍스트 기반 디자인

## 3. 기술 스택

| 구분 | 기술 | 선정 이유 |
|------|------|-----------|
| Framework | **Next.js 15 (App Router)** | 정적 사이트 생성(SSG), SEO 최적화 |
| 스타일링 | **Tailwind CSS** | 빠른 스타일링, 모던한 디자인 시스템 |
| 폰트 | Pretendard + Inter | 한/영 가독성 최적 조합 |
| 배포 | Vercel | Next.js 최적 배포 플랫폼 |

## 4. 페이지 구조

```
/ (Single Page)
├── Header (이름, 직함, 연락처)
├── CareerSummary (경력 요약 3~4줄: 연차, 전문 분야, 핵심 성과)
├── Skills (기술 스택 뱃지)
├── Experience (경력 타임라인)
├── Projects (프로젝트 카드)
├── Education (학력/자격증)
└── Footer (링크, 저작권)
```

## 5. "일 잘하는 사람" 어필 전략

### 면접관 관점에서의 설계
1. **성과 중심 기술**: "~을 개발" 대신 "~을 개발하여 성능 30% 개선" 형태
2. **수치화**: 가능한 모든 성과를 숫자로 표현
3. **문제 → 해결 구조**: 프로젝트마다 어떤 문제를 어떻게 해결했는지 명시
4. **기술 스택 명확화**: 실무에서 사용한 기술과 수준을 한눈에 파악
5. **깔끔한 코드 = 깔끔한 이력서**: 정돈된 레이아웃 자체가 개발자의 역량을 보여줌

## 6. 데이터 관리

이력서 내용은 별도 JSON/TS 데이터 파일로 분리하여 관리:

```
src/
├── data/
│   └── resume.ts          # 이력서 데이터 (타입 안전)
├── components/
│   ├── Header.tsx
│   ├── CareerSummary.tsx
│   ├── Skills.tsx
│   ├── Experience.tsx
│   ├── Projects.tsx
│   ├── Education.tsx
│   └── Footer.tsx
└── app/
    ├── layout.tsx
    └── page.tsx
```

## 7. 마일스톤

| 단계 | 내용 | 산출물 |
|------|------|--------|
| M1 | 프로젝트 셋업 + 데이터 구조 정의 | Next.js 프로젝트, resume.ts |
| M2 | 레이아웃 + 컴포넌트 구현 | 전체 섹션 컴포넌트 |
| M3 | 스타일링 + 반응형 | Tailwind 스타일 완성 |
| M4 | 최종 검토 + 배포 | Vercel 배포 완료 |

## 8. 성공 기준

- [ ] 면접관이 30초 안에 핵심 역량을 파악할 수 있는가
- [ ] 모바일에서도 가독성이 유지되는가
- [ ] 데이터 수정만으로 내용 업데이트가 가능한가
- [ ] Lighthouse Performance 90+ 달성
- [ ] 프린트 시에도 깔끔한 레이아웃인가
- [ ] PDF 이력서와 동일한 내용 구조를 유지하는가
- [ ] PDF 다운로드 버튼이 정상 동작하는가
